# 港討 HKHS — 小港高中匿名討論區

專屬小港高中學生的匿名討論區。取代以往「私訊 IG → 版主截圖 → 上傳」的流程，
發文即時、留言即時，而且**貼文可以搜尋**。

## 目前進度：主視覺介面原型（假資料）

- `index.html` — 頁面結構
- `style.css` — 主視覺（簡潔圓潤風格、海港藍綠主色、深淺色模式）
- `app.js` — 互動邏輯與假資料（`POSTS` 陣列）

### 本機預覽

```
python -m http.server 8321
```

打開 http://localhost:8321 （建議用瀏覽器的手機模擬模式）

### 已完成的功能（前端）

- 手機優先版面（桌機自動置中 520px）
- 貼文流：全部 / 看板篩選、最新排序
- 熱門頁：依讚數＋留言數排序
- **關鍵字搜尋**：標題、內文、留言全文搜尋＋關鍵字標亮
- 貼文詳情頁：完整內文、樓層留言（B1、B2…）、即時留言
- 匿名發文：底部彈窗、隨機海洋生物身分（可換）、選看板
- 按讚、深淺色切換（記憶偏好）、Toast 提示
- **版主隱藏入口**：`/admin.html`（主站沒有任何登入按鈕，只有版主知道網址）
  - 原型階段通行碼寫在 `admin.html` 裡（`kgsh-demo`），僅供展示
  - 登入後主站頂部顯示「⚓ 版主模式」，貼文與留言出現刪除鈕
  - 正式上線時改用 Supabase Auth 驗證，通行碼機制即移除
- **版主管理功能**（登入後）：
  - 發文時可切換「以版主身分發文」（顯示 📣 版主 ⚓，不再匿名）
  - 置頂／取消置頂貼文（📌，置頂貼文排在首頁最前）
  - 編輯看板：新增、改名、改圖示與說明、刪除（板內貼文自動移到其他看板），存於 localStorage
  - 編輯「關於」頁各段落內容（含版規），存於 localStorage
- **管理後台**：[dashboard.html](dashboard.html)（版主登入後點頂部「管理後台」進入，非版主會被導回登入頁）
  - 總覽數字：總貼文數、總留言數、今日新增、隱藏中數量
  - 看板分佈長條圖
  - **檢舉佇列**：所有被檢舉過的貼文與留言集中列出（含還沒到自動隱藏門檻的），
    一次看完不用滑遍整條動態去找，可直接恢復顯示或刪除
  - 看板／關於頁編輯的捷徑連結

### 社群自治機制（十年免人工整理的核心）

- **檢舉 → 自動隱藏**：任何人可檢舉貼文／留言，累積 3 次自動隱藏，
  不需要版主動手。版主可查看被隱藏的內容（⚠️ 標示），
  人工「恢復顯示」（貼文與留言皆可）或刪除
- **公告看板僅限版主**：學生發文時選不到「公告」，只有版主能發公告
- **發文冷卻**：發文 60 秒、留言 15 秒（版主豁免），防洗版
- **熱門時間衰減**：熱門榜分數隨時間衰減（類 Hacker News），舊文不會永遠霸榜
- **分頁載入**：貼文每頁 20 篇、「載入更多」按鈕，十年貼文量也不怕
- **貼文專屬連結**：`#post/123` 可直接分享（支援手機原生分享面板→IG／LINE），
  瀏覽器返回鍵可關閉貼文
- **字數上限**：標題 40、內文 2000、留言 300

## 資料庫（Supabase 免費方案）——已串接完成

程式碼支援雙模式：`config.js` 沒填金鑰 = **展示模式**（假資料，重整就還原）；
填了金鑰 = **正式模式**（所有資料存 Supabase，即時生效）。

### 上線三步驟（版主做一次，約 10 分鐘）

1. **建專案**：到 <https://supabase.com> 用信箱註冊 → New project
   （名稱隨意、密碼記好、區域選 Northeast Asia (Tokyo)）
2. **貼 SQL**：左側 SQL Editor → New query → 貼上本資料夾
   [schema.sql](schema.sql) 的全部內容 → Run
   （建表、安全規則、預設看板與公告一次完成）
3. **設定金鑰與版主帳號**：
   - Authentication → Users → Add user：建立版主的信箱＋密碼
   - Authentication → Sign In / Providers → Email →
     **關閉「Allow new users to sign up」**（必做！不關任何人都能變版主）
   - ⚙️ Project Settings → **Data API**：複製 Project URL；
     ⚙️ Project Settings → **API Keys** → Publishable and secret API keys 分頁：
     複製 **Publishable key**（`sb_publishable_` 開頭）——兩個值貼進
     [config.js](config.js)（舊專案改用 Legacy API Keys 分頁的 anon public）

之後把整個資料夾部署到 **Cloudflare Pages / Netlify / Vercel**（皆免費、
拖拉資料夾就能上線），版主登入 `你的網址/admin.html`（信箱＋密碼）。

### 安全設計（規則都在資料庫端，前端擋不住的這裡擋）

- 匿名者**不能直接寫任何資料表**，只能呼叫 4 個資料庫函式：
  發文、留言、按讚、檢舉——每個函式內建自己的規則
- 檢舉滿 **3 個不同裝置**自動隱藏（跨裝置計數，伺服器端執行）
- 發文 60 秒／留言 15 秒冷卻（伺服器端計時，繞過前端也沒用）
- 公告看板限版主、標題／內文／留言長度上限：資料庫端再驗一次
- 隱藏的貼文與留言，未登入者**連讀都讀不到**（RLS）
- 版主的置頂／隱藏／恢復／刪除／編輯看板與關於：需要登入（Supabase Auth）

## 分享功能（連結預覽＋限時動態分享圖）

貼文分享連結從 `#post/123`（純前端 hash，機器人抓不到內容）改成真正的路徑
`/p/123`，搭配兩支伺服器端函式解決「分享到 IG 只有連結、看不到內容」的問題：

- [api/post.js](api/post.js)：真人點擊 `/p/123` 會自動導回站內對應貼文；
  IG／LINE／FB 等平台的預覽機器人抓到的則是這篇貼文專屬的標題、摘要、圖片
  （被隱藏或已刪除的貼文會回傳網站預設的通用預覽，不會外流）
- [api/og.js](api/og.js)：用 [@vercel/og](https://www.npmjs.com/package/@vercel/og)
  即時產生分享圖，同一支函式依 `format` 參數輸出兩種尺寸：
  - `format=card`（1200×630）：連結預覽縮圖
  - `format=story`（1080×1920）：限時動態卡片，模仿「轉發貼文截圖」的質感，
    logo／標語／貼文卡片／網址都在 IG 限動會被介面遮住的安全區以內
- 詳情頁的「分享到限時動態」鈕：跟伺服器要一張對應貼文的限動圖 →
  叫出手機原生分享面板（面板裡本來就會列出「Instagram 限時動態」選項，
  點一下就帶圖進去，不需要網站自己做不穩定的深層連結）

### 這兩支函式需要的環境變數（跟 config.js 用同一組公開值即可）

到 Vercel 專案設定 → Environment Variables，新增（development／preview／
production 三個環境都要）：

- `SUPABASE_URL` — 跟 [config.js](config.js) 裡的 `url` 一樣
- `SUPABASE_ANON_KEY` — 跟 [config.js](config.js) 裡的 `anonKey` 一樣

本機開發可以用 `vercel env pull .env.local` 直接抓下來，不用手動複製貼上。

  版本固定不會自己更新壞掉；想完全離線可下載該檔改成本地引用
- **版主交接**：畢業前把 Supabase 帳號與版主信箱密碼交給下一屆即可

### 誠實聲明：無法 100% 自動化的部分

- 免費平台的**政策變動**（額度調整、服務收掉）——選大廠（Cloudflare／Vercel／Supabase）風險最低，但十年內仍可能需要搬家一次
- **重大濫用事件**（觸法內容、報警處理）仍需要有人出面
- 檢舉門檻可能被惡意濫用（惡意檢舉正常貼文）——版主偶爾登入按「恢復顯示」即可
